# Pain Mine

Two modes. State which you're running.

## Mode A — Vertical pain (Stage 1)
Find the most acute, expensive, frequent problem an owner in a vertical has TODAY.

1. Search the vertical's owner communities, Reddit, forums, and "biggest challenges running a <vertical>" content.
2. For each candidate pain, quantify: $ lost per occurrence × frequency × do they already pay to fix it.
3. Score **Pain Intensity 0–10** = acute × expensive × frequent × already-paying.
4. **Gate:** ≥7 ("hair on fire") to proceed. Vitamins (nice-to-have) die here.

## Mode B — Incumbent weakness (Stage 4)
Find the repeated, ownable failure of the leading players — the wedge.

1. Pull 50+ reviews across G2, Capterra, Trustpilot + Reddit threads for the incumbent(s) (WebSearch + WebFetch the review pages).
2. Cluster complaints. Look for the *repeated* one (e.g., Weave: "support vanishes after onboarding"; Smith.ai: "billing 20–30% over base"). One-off gripes don't count.
3. Identify which weakness we could build positioning around AND credibly deliver better (reliability, support, transparent billing, Canada/accent, integration depth).

## Output
- Mode A: ranked pain list with intensity scores + the one hair-on-fire pain.
- Mode B: the top 2–3 repeated weaknesses with representative quotes + the positioning opening.
- Save as `pain_<slug>.md`; run `/research-brain` to update node 1 (or node 2 for incumbent weakness).

## Honesty rules
- Quote real reviews; don't invent. If review volume is thin, say so — a weakness seen 3× isn't yet a pattern.
- Separate "users complain because the category is hard" from "this vendor specifically fails." Only the latter is an ownable wedge.
